## 一、模块概述

第三期在任务看板核心功能基础上，新增**个人数据统计页**，以 2×2 网格布局展示 4 个图表，帮助用户快速了解任务分布情况。

## 二、功能需求

### 2.1 数据可视化

**用户故事**：作为用户，我希望看到任务的统计图表，了解自己的任务分布情况。

**功能描述**：
- 使用 Recharts 库展示统计图表
- 包含以下 4 个图表：
  - **状态分布**（环形图）：按看板列分组计数，中心显示任务总数
  - **优先级分布**（环形图）：按高/中/低分组计数，中心显示任务总数
  - **标签分布**（横向柱状图）：按标签分组计数，柱子用标签自身颜色着色
  - **状态数量**（横向柱状图）：每根柱子对应一个看板列，直观对比数量差异
- 仅展示当前用户的个人数据
- 页面布局为 2×2 网格（桌面端），移动端自动折叠为单列
- 无任务时显示空状态提示

**数据隔离**：
- 所有统计查询严格按 `created_by=当前用户` 过滤
- 状态 label 从用户的 BoardColumn 获取（匹配 slug）
- 标签数据从用户的 Tag 获取，颜色取自 Tag.color 字段
- 只统计未删除的任务（`is_deleted=False`）

**后端实现**：
- 新增专用统计 API：`GET /api/statistics/`
- 使用 Django ORM 聚合（`annotate(Count)`），不拉取原始任务数据
- 通过 `drf-spectacular` 自动生成 OpenAPI schema

**前端实现**：
- 新增统计页路由：`/statistics`
- 侧边栏新增「统计」导航入口
- 4 个独立图表组件 + 1 个空状态组件
- 支持中英文 i18n

## 四、验收标准

### 4.1 数据可视化

- [ ] 侧边栏显示「统计」导航入口
- [ ] 点击进入统计页，展示 2×2 网格布局
- [ ] 状态分布环形图正确展示各列任务数量，中心显示总数
- [ ] 优先级分布环形图正确展示高/中/低任务数量
- [ ] 标签分布横向柱状图正确展示各标签任务数，柱子颜色与标签颜色一致
- [ ] 状态数量横向柱状图正确展示各列任务数
- [ ] 无任务时显示空状态提示，不渲染图表
- [ ] 桌面端 2×2 网格，移动端自动折叠为单列
- [ ] 所有查询严格按 `created_by=request.user` 过滤，不会出现串数据
- [ ] 自定义列改名后，统计页 label 同步更新
- [ ] `openapi:generate` 执行后前端类型自动更新
