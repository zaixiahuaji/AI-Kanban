## 一、模块概述

---

## 二、功能需求

### 2.1 附件上传

**用户故事**：作为用户，我希望在任务中上传附件（图片、文档等），方便管理任务相关的文件。

**功能描述**：
- 每个任务支持上传附件
- 单个文件最大 5MB
- 每个任务最多 5 个附件
- 支持常见格式（图片、文档、压缩包）
- 仅上传者可下载和删除自己的附件
- 文件存储在服务器本地磁盘

**校验规则**：
- 前端：文件类型校验、大小校验、数量校验
- 后端：文件类型白名单（jpg/png/gif/pdf/doc/xls/zip 等）、大小限制、数量限制

---

### 2.2 纯文本评论

**用户故事**：作为用户，我希望在任务下添加评论，记录想法和讨论。

**功能描述**：
- 每条任务下可添加文本评论
- 评论按时间正序排列展示
- 评论者可以删除自己的评论
- 任务创建者可以看到所有评论
- 管理员可删除任意评论

**校验规则**：
- 前端：评论内容非空、长度限制
- 后端：权限校验（只能删除自己的评论，管理员除外）

---

### 2.3 站内通知

**用户故事**：作为用户，当有人在我的任务下评论时，我希望收到通知。

**功能描述**：
- 触发场景：有人在我的任务下发表评论
- 通知方式：站内通知（小红点 + 通知列表）
- 用户点击小红点后展开通知列表，查看未读通知
- 通知可标记为已读
- 评论时实时写入数据库，用户刷新页面后可见

**不在范围内**：
- 邮件通知
- 浏览器推送（Push API）
- WebSocket 实时推送

---

### 2.4 数据可视化

**用户故事**：作为用户，我希望看到任务完成的统计图表，了解自己的工作效率。

**功能描述**：
- 使用 Recharts 库展示统计图表
- 包含以下图表：
  - 任务完成率（饼图或环形图）
  - 每日任务统计（折线图或柱状图）
- 普通用户查看个人统计
- 管理员可查看全站统计

## 四、验收标准

### 4.1 附件上传

- [ ] 可在任务中上传文件
- [ ] 单文件超过 5MB 时提示错误
- [ ] 超过 5 个附件时提示错误
- [ ] 可下载附件
- [ ] 仅上传者可删除附件

### 4.2 评论

- [ ] 可在任务下添加文本评论
- [ ] 评论按时间正序显示
- [ ] 评论者可删除自己的评论
- [ ] 管理员可删除任意评论

### 4.3 通知

- [ ] 评论创建后，任务所有者收到通知
- [ ] 通知列表显示未读通知，带小红点
- [ ] 可标记通知为已读
- [ ] 刷新页面后新通知可见

### 4.4 数据可视化

- [ ] 显示任务完成率图表
- [ ] 显示每日任务统计图表
- [ ] 普通用户看个人数据，管理员看全站数据
