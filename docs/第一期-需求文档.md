## 一、项目背景

基于 Turbo 脚手架（Django + Next.js）进行二次开发，构建一个**任务看板管理系统**。系统面向个人和小团队，提供任务管理、团队协作、数据可视化等能力。

第一期聚焦**基础能力建设**：国际化、邮箱验证码注册、角色权限。为后续任务看板核心业务打基础。

### 分期规划总览

| 期数 | 主题 | 包含功能 |
|------|------|----------|
| 第一期 | 基础能力 | 中英文切换、邮箱验证码注册、RBAC 角色权限 |
| 第二期 | 核心业务 | 任务看板 CRUD、用户自定义标签、逻辑删除 + 回收站 |
| 第三期 | 扩展功能 | 附件上传、纯文本评论、站内通知、数据可视化 |

## 二、功能需求

### 2.1 中英文语言切换

**用户故事**：作为用户，我希望在页面上手动切换中/英文，系统记住我的选择，下次访问时保持该语言。

**功能描述**：

- 页面提供语言切换入口（如下拉菜单或图标按钮），可选"中文"和"English"
- 切换后，当前页面及后续所有页面立即切换为目标语言
- 语言偏好持久化存储，关闭浏览器后重新访问仍保持
- 切换范围覆盖：
  - 前端所有页面文本（导航、按钮、表单标签、提示信息等）
  - Django Admin 后台界面
  - 后端 API 返回的错误消息（如"用户名已存在"）

**不在范围内**：

- 自动检测浏览器语言
- 多于两种语言的扩展（本期只做中英文）

---

### 2.2 邮箱验证码注册

**用户故事**：作为新用户，我希望在注册时填写邮箱并验证，注册成功后可以直接登录使用，无需管理员手动激活。

**功能描述**：

- 注册页面包含以下字段：用户名、邮箱、密码、确认密码、验证码
- 邮箱字段旁边有"发送验证码"按钮，用户填写邮箱后点击发送
- 系统向该邮箱发送 6 位数字验证码，有效期 5 分钟
- 发送后按钮进入 60 秒倒计时，倒计时结束前不可重发
- 用户填写所有字段后点击"注册"，系统验证验证码正确后创建账号
- 注册成功的账号状态为直接可用，无需管理员手动激活
- 废除原有的"注册后需管理员激活"流程

**校验规则**：

- **前端校验**：
  - 邮箱格式校验
  - 密码长度校验（≥8 位）
  - 确认密码一致性校验
  - 验证码非空校验
- **后端校验**：
  - 邮箱唯一性校验（一个邮箱只能注册一个账号）
  - 用户名唯一性校验
  - 密码强度校验（≥8 位）
  - 验证码正确性及有效期校验（5 分钟内）
  - 验证码使用后标记为已使用，防止重复使用

**防刷机制**：

- 同一邮箱 60 秒内限发一次验证码
- 同一邮箱 24 小时内限发 10 次验证码

**不在范围内**：

- 邮箱变更验证
- 密码找回流程

### 2.3 RBAC 角色权限

**用户故事**：作为管理员，我可以管理所有用户和查看所有任务；作为普通用户，我只能操作属于自己的数据。

**角色定义**：

| 角色 | 权限范围 |
|------|----------|
| 管理员 | 查看所有用户的任务、在 Admin 后台管理用户（启用/禁用/删除）、查看全站统计数据 |
| 普通用户 | 只能查看/操作自己的任务和账号信息 |

**功能描述**：

- 系统区分两种角色：管理员和普通用户
- 普通用户只能查看和操作自己创建的数据（任务、附件、评论等）
- 管理员可以查看所有用户的数据
- 用户管理（查看用户列表、启用/禁用账号）通过 Django Admin 后台完成
- 前端界面根据用户角色展示不同的导航和操作入口（如管理入口仅管理员可见）
- 前端权限仅为展示层优化，实际权限校验在后端执行

**不在范围内**：

- 自定义角色系统
- 权限粒度配置
- 前端独立的用户管理页面

## 三、非功能需求

### 3.1 性能需求

- 页面切换流畅，无明显白屏或卡顿
- 接口响应时间 ≤ 300ms（正常负载下）
- 验证码发送响应时间 ≤ 3 秒（取决于邮件服务）
- 分页加载，避免大数据量一次性返回

### 3.2 安全需求

- 用户密码加密存储，禁止明文
- 细粒度权限控制，禁止越权访问他人任务/数据
- 验证码防刷：同一邮箱 60 秒内限发一次，24 小时内限发 10 次
- 文件上传格式/大小限制（第三期实施，此处先定义原则）

### 3.3 兼容性

- 支持主流浏览器最新两个大版本（Chrome、Firefox、Edge、Safari）
- 响应式布局，适配桌面端（≥1024px）和平板端（≥768px）

## 五、验收标准

### 5.1 中英文切换验收标准

- [ ] 页面右上角有语言切换入口，可选中文/English
- [ ] 切换后所有页面文本立即变为目标语言
- [ ] 关闭浏览器重新打开，语言保持上次选择
- [ ] Django Admin 界面支持中英文
- [ ] API 错误消息支持中英文

### 5.2 邮箱验证码注册验收标准

- [ ] 注册页面包含：用户名、邮箱、密码、确认密码、验证码字段
- [ ] 点击"发送验证码"后收到邮件，邮件包含 6 位数字验证码
- [ ] 60 秒倒计时期间不可重发
- [ ] 验证码 5 分钟内有效，过期需重新发送
- [ ] 使用正确验证码注册成功后，账号可直接登录
- [ ] 重复邮箱注册失败并提示
- [ ] 前后端校验均生效

### 5.3 RBAC 角色权限验收标准

- [ ] 管理员可查看所有用户的任务
- [ ] 普通用户只能看到自己的任务
- [ ] 管理员可在 Django Admin 中管理用户
- [ ] 普通用户无法访问 Admin 后台
- [ ] 前端根据角色显示不同的导航入口
- [ ] 直接调用 API 绕过前端时，后端权限仍然生效
